【PWA 静的ホスティング用デプロイパッケージ】
こちらのZIPファイルを解凍し、そのマウントディレクトリ内のすべてのファイルをホスティングサーバー
（Netlify, Vercel, Firebase Hosting, GitHub Pages、または通常のレンタルサーバー等）に
そのままアップロードしてください。

■ 重要：ローカルPCで確認する場合の注意（表示が崩れる・動かない場合の解決方法）
Viteでビルドされた成果物（PWA）は、最新のブラウザ仕様に準拠しており、
「ES Modules (type="module")」および「CORS（セキュリティ機能）」を使用しています。

そのため、以下の2点に必ずご注意ください：

1. 【ZIPファイルは必ず解凍（展開）してください】
   ZIPファイルを解凍せずにエクスプローラー内から直接「index.html」をダブルクリックして開いた場合、
   同じフォルダーにあるCSSファイルやアイコン画像が正しく読み込まれず、表示が崩れてしまいます。
   右クリックして「すべて展開」などを行ってからご使用ください。

2. 【ダブルクリック（file:// プロトコル）では動作しません】
   解凍後であっても、「index.html」をダブルクリックで直接ブラウザに読み込ませた場合（アドレスバーが file:///... となっている状態）、
   ブラウザのセキュリティ上の制限（CORSポリシー制限）によってCSSおよびJavaScriptの読み込みが完全にブロックされます。
   
   動作確認や本番運用を行う際は、必ず「Webサーバー」を介して「http://」または「https://」のアドレスでアクセスしてください。
   ・ローカルPCでの確認方法：VS Codeの「Live Server」拡張機能や、コマンドライン等で「npx serve」「python -m http.server 8000」などの簡易ローカルWebサーバーを起動してアクセスしてください。
   ・本番公開：Netlify, Vercel, レンタルサーバー等のホスティング環境に解凍後のファイル群をそのまま配置してください。

■ 逆ジオコーディング（住所の自動取得）について
当アプリは、緯度経度から日本語の住所を特定するためにAPIサーバーを利用しています。
本パッケージは静的なアセットのみで構成されていますが、ご自身の静的ホスティングサーバーに設置した場合でも、
アプリの右上、またはダウンロードエリアに用意されている「APIサーバー接続先設定」から、
開発元のサーバーURL、もしくはご自身で立ち上げたAPIサーバーのURLを設定することで、
逆ジオコーディングの住所取得機能を100%継続して動作させることが可能です。

API URL 設定の記述例:
https://[元のアプリケーションドメイン]/api/geocode

■ PWAのインストールとオフライン機能
本パッケージは、PWAマニフェスト (manifest.json) 、最新のサービスワーカー (sw.js)、および
アプリ用アセットアイコンが設定されています。
HTTPS（常時SSL）対応のサーバーにアップロードすることで、PCやスマートフォンへ「アプリとしてインストール」し、
いつでもすぐにデスクトップから軌跡から高度計算やフライトログ可視化などをオフラインで高密度に動作させることができます。
