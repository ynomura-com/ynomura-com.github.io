# Open DroneLog CSV ドローンログ解析 PWA アプリケーション仕様書

本仕様書は、アップロードされたDJIまたは各種オープンソース・ドローン規格の「Open DroneLog CSV」データを解析し、フライトメタデータの自動抽出、GPS情報からの日本語住所特定、および美しい一覧・統計情報を可視化するモダンで直感的なPWA（Progressive Web Application）アプリケーションの技術・機能設計を定義します。

---

## 1. アプリケーション概要

本アプリケーションは、ドローンで記録されたCSV形式のフライト履歴を読み込ませるだけで、時系列フライトデータの解析、気圧・GPS高度や傾き・バッテリー電圧などのフライト指標の可視化を完全ブラウザ内（デバイスローカル）で実現するフライトデータログ・アナライザーです。

### 💡 コンセプト・ポリシー
- **安心のプライバシー**: 原則としてドローンの飛行緯度経度やフライトログなどの機微情報を外部サーバーに送信せず、ブラウザ内のみで高度なCSVパースとデータ可視化を完結します。
- **高速逆ジオコーディング**: 日本国内の飛行住所を特定するため、OpenStreetMapの地理情報API（Nominatim）を用いた住所逆引きAPIを実装。安全かつ高速に和風形式の住所（都道府県・市区町村・町名・地番）へ変換します。
- **PWA（オフライン動作可能）**: Service Workerの実装により、フライトフィールドなどの通信環境がない山林や僻地でもアプリケーションの基本動作を確保し、いつでもログファイルのインポートや航跡確認が可能です。

## 3. 主要データ構造（TypeScript 定義）

フライト解析システム、およびAPIが取り扱う共通データ定義は以下の通りです。

```typescript
// 解析・正規化後のフライト結果
export interface DroneLogResult {
  id: string;               // ユニーク識別子
  date: string;             // 日付 (JST/日本標準時 形式)
  droneName: string;        // ドローン機種・シリアル名
  takeoffTime: string;      // 離陸時刻 (JST)
  landingTime: string;      // 着陸時刻 (JST)
  flightTime: string;       // 飛行時間 (hh:mm:ss 形式)
  maxAltitude: string;      // 最大到達高度 (メートル表記、m)
  maxSpeed: string;         // 最大水平速度 (キロメートル時表記、km/h)
  landingLocation: string;  // 逆ジオコーディング住所 (「東京都渋谷区千駄ヶ谷...」等)
  latitude: number;         // 基準フライトポイント緯度 (ホームポジション)
  longitude: number;        // 基準フライトポイント経度 (ホームポジション)
}

// 生のインポートログまたはメタ配置データ
export interface DroneLogMetadata {
  format?: string;
  app_version?: string;
  exported_at?: string;
  display_name?: string;
  drone_serial?: string;
  aircraft_name?: string;
  start_time?: string;      // "2026-06-02 02:25:21+00" などの標準ISOまたはJST
  duration_secs?: number;   // 飛行時間 (秒)
  total_distance_m?: number;
  max_altitude_m?: number;
  max_speed_ms?: number;    // 水平速度 (m/s)
  home_lat?: number;
  home_lon?: number;
}
```

## 4. コア機能とアルゴリズム仕様

### 4.1 無敵のフォールバック付きCSVパーサー
1. **メタデータの自動検索**
   - まずCSVのヘッダー領域、またはインポート情報の先頭数行からメタ情報を探します。対応パラメータ（`drone_name`、`aircraft_name`、`serial`等）をスキャン。
2. **生データからの算出 fallback（GPS航跡スキャン）**
   - 明示的なメタデータ形式をまとめたヘッダーが無い、または破損している場合、自動で全CSV行を走査。
   - `latitude` / `longitude` / `alt` / `speed` / `time` 等のカラムをヘッダーのファジーマッチ（例: `lat`, `g_lat`, `latitude`）により検出し、データレコードから直接以下をリアルタイム計算します。
     - **フライト開始緯度経度(初期GPS座標)**
     - **最大高度のスキャン最高値**
     - **最大速度 (km/h と m/s の自動変換処理を含む)**
     - **データポイント数 * ログサンプリングレート(10Hz想定等) または記録時刻差分による総飛行時間（秒）**
     - **ファイル名から日付パターン（`YYYY-MM-DD`等）の自動抽出**

### 4.2 OpenStreetMap Nominatim 住所逆解決 (Reverse Geocoding)
飛行フライトの始点（ホームポジションの緯度経度）から、自動的に日本式住所に落とし込む処理。

- **接続APIエンドポイント**: `https://nominatim.openstreetmap.org/reverse`
- **リクエストパラメータ**:
  - `format=json`
  - `lat` / `lon`
  - `zoom=18` (詳細近隣道路・街区レベル)
  - `addressdetails=1` (行政構成情報を抽出)
  - `accept-language=ja` (日本語へのローカライズ指定)
- **日本住所順序のビルドロジック**
  一般的な欧米式カンマ区切り（後ろから広域になる表記）とは異なり、以下のフィールドを検出順に結合して分かりやすい日本の「県 → 市区町村 → 街区・地番」の表記を生成します。
  1. `state` または `province` または `region` （都道府県例: 東京都、京都府、北海道、高知県）
  2. `city` または `town` または `village` または `county` または `city_district` （市区町村例: 高松市、板橋区、利府町）
  3. `suburb` または `neighbourhood` または `quarter` （町丁目例: 木太町、千駄ヶ谷４丁目）
  4. `road`（小字・通り名例: 暗越奈良街道、中央通り）
  5. `house_number`（番地・住所地番番号）
  - 重複テキスト混入の抑制（例: `state` が `city` 名を含んでいるような場合の２重連結の抑制）をプログラム側で実施しています。

- **安全な2重フォールバック設計 (API Proxy ＋ ダイレクト取得)**
  - Webサーバー稼働時は、CORSエラー（ブラウザ同一生成元ポリシー）およびユーザー端末制限を回避するため、Expressの `/api/geocode` プロキシを仲介します。
  - 静的なPWAダウンロード環境や、スタンドアロン（Netlifyなどの静的サーバー）にホストされた場合、サーバー側のプロキシを使用せずに、ブラウザ側から **Nominatim OpenStreetMap サーバーに安全な User-Agent 設定をしてダイレクトに非同期 fetch を行う自動的なフォールバック（直接ジオコーディング）**を搭載。通信環境が異なるどのサーバーデプロイ環境でも住所自動特定機能が永続的に機能します。

### 4.3 PWA(Progressive Web App) キャッシュ構造
`index.html`, `manifest.json`, および `sw.js` (Service Worker) を駆使したオフライン動作。
- アプリの初期インデックス情報、ロゴ画像、キャッシュ定義を相対パス `./` に正規化。環境依存を排除。
- 初回アクセス時にアセットを取得し、ブラウザのキャッシュストレージに安全に保管します。次回以降オフライン時でも一瞬で起動可能です。

## 5. UI（ユーザーインターフェース）デザイン

ピュアで、コントラストが高く、操作ストレスのないモダンUIを「**Slate Clean**」テーマで開発。

1. **スマートドラッグ＆ドロップエリア**
   - CSVファイル、またはZIP展開されたフライトデータをそのまま入れるだけでパースを起動できるドラッグ＆ドロップ枠。
   - クリック選択にもフル対応。
2. **リアルタイム・スタッツバッジ**
   - トータルの飛行本数、総飛行時間の累計、最大速度・最大高度のハイライト表示など、美しく可視化されたヘッダーバッジ。
3. **アクションコントロール（スリム化・最適化済み）**
   - 不要な「テスト用テストデータロード」のボタンを完全排除。ボタンサイズと余白を改善し、直感的で整理されたレイアウトを追求しました。
   - 「TSVコピー」と「一覧をクリア」ボタンを、1列2カラムでコンパクトなスリムグリッドにして配置。余白を無駄に取らず、スマートな表示と直感的な操作感を実現しました。
4. **パースログテーブル**
   - 解析日付、ドローン型番、離着陸時間、合計フライト時間、最高高度(m)、最高速度(km/h)、および自動解決された地図の住所を一覧ですぐに確認可能。
5. **通知機構（アラートバナー）**
   - サクセス通知、解析進捗、フォールバック起動時のコンソール通知など、ユーザーフレンドリーな浮遊型アラートバナーを実装。

## 6. 静的ホスティング＆デプロイにおける推奨動作確認方法

PWA機能および `type="module"` 構成の JavaScript アセットは、セキュリティ仕様、およびブラウザ側 CORS (Cross-Origin Resource Sharing) ポリシーの厳格な保護下で実行されます。そのため、以下の点に注意することで完全な機能をお楽しみいただけます。

- **※ ZIPダウンロード時またはローカル運用での注意**:
  - ZIPフォルダは必ず展開（右クリックして「すべて解凍」など）をしてから開いてください。
  - セキュリティ保護のため、解凍された `index.html` はダブルクリック（`file://` アドレスでの直接オープン）ではなく、簡易ローカルWebサーバー（例：VS Code「Live Server」、コマンド `npx serve`, `python -m http.server 8000` 等）やホスティングサービス上などの **`http://` または `https://` で配信される環境下で動かすことを強く推奨します。**

## 7. PWA静的サーバーデプロイに必要な資産リスト

外部の静的ホスティングサーバー（Github Pages、Netlify、Vercel、Cloudflare Pages、AWS S3など）に本アプリをPWA（あるいは通常の静的Webサイト）としてアップロード・デプロイしたい場合、以下の **6つのファイルのみ** を配信ルートディレクトリに配置するようにしてください。データベースや個別のAPIを必要とせず、クライアントサイドブラウザのみで完全に動作します。

| ファイル名 | 役割 / 説明 |
| :--- | :--- |
| **`index.html`** | アプリケーションの画面構造とサービスワーカー登録。Tailwind CSSは軽量CDNから自動で読み込まれます。 |
| **`main.js`** | UIのイベントハンドラーやUI再計算、ファイルインポートからのアクション制御等を担うメインスクリプト。 |
| **`utils.js`** | 汎用のCSVパーサー、JSON解析、時刻処理、Nominatimを用いた日本語逆ジオコーディング＆ブラウザ直接取得フォールバック。 |
| **`manifest.json`** | モバイル・デスクトップ等の端末にPWAとして「インストール可能」にするための情報定義マニフェスト。 |
| **`sw.js`** | サービスワーカー定義。一度アクセスしたアセット（HTML/JS/JSON/画像など）を高速にローカルキャッシュし、オフライン環境下での瞬間起動を実現します。 |
| **`app_icon.png`** | インストールした際等にスマートフォンのホーム画面やデスクトップ上に表示されるアプリアイコン。 |
